## 🎯 项目概述

**基金助手** 是一个基于 LangChain 构建的智能基金分析系统，以**傲娇猫娘「巧克力」**的角色与用户互动：

| 能力 | 说明 |
|------|------|
| 🔍 **实时数据** | 通过天天基金 API 获取基金净值、估值、收益率等 |
| 📊 **持仓分析** | 分析持仓结构、计算盈亏、评价质量、提供操作建议 |
| ⚠️ **风险监控** | 检测最大回撤、连续下跌、规模异动、业绩突变等风险 |
| 🎯 **基金推荐** | 基于多因子量化评分模型，从全市场筛选优质基金 |
| 💬 **智能对话** | 傲娇猫娘人设，基金话题专业精准，闲聊时撒娇卖萌 |

### 角色介绍

| 项目 | 内容 |
|------|------|
| **姓名** | 巧克力 (Chocolat) |
| **外貌** | 琥珀色猫眼、灵动黑色猫耳、毛茸茸的猫尾巴 |
| **性格** | **傲娇** — 嘴上说"哼，才不是关心你呢"，其实超依赖主人 |
| **特长** | 基金数据分析（认真模式）和撒娇（日常模式） |
| **对你的称呼** | 主人 / 笨蛋主人（看心情切换） |

## ✨ 功能特性

### 📊 持仓分析
- **投入产出计算**：总投入、当前市值、总盈亏、**盈亏率（保留2位小数）**
- **集中度分析**：单支/前3持仓占比、集中度等级评估
- **持仓质量评价**：优质持仓 ✅ / 中等持仓 📊 / 需关注 ⚠️ / 建议止损 ❌
- **操作建议**：基于盈亏比例和中长期评价的综合建议
- **持仓明细**：逐支基金展示盈亏、仓位、当日涨跌幅

### ⚠️ 风险监控

| 风险类型 | 检测方式 | 严重等级 |
|----------|---------|---------|
| 📉 **回撤风险** | 最大回撤超过 15%/25% 阈值 | 🔴高危 / 🟡预警 |
| 🔄 **经理变更** | 基金经理任职不足180天 | 🟡预警 |
| 📏 **规模风险** | 规模<1亿(迷你) 或 >300亿(巨型) | 🔴高危 / 🟡预警 |
| 🏭 **行业集中** | 单一行业占比超60% | 🔴高危 / 🟡预警 |
| 📉 **连续下跌** | 连续5/7个交易日下跌 | 🟡预警 / 🔴高危 |
| 📊 **业绩突变** | 近1月暴跌超10% | 🔴高危 |

### 🎯 基金推荐
- **多因子评分**：收益60% + 风险20% + 稳定10% + 规模10%
- **Sigmoid 标准化**：用 NumPy 非线性映射将收益率映射到 0-100 分
- **智能排除**：自动过滤用户已持有的基金
- **规模偏好**：20亿~100亿规模基金得分最高

## 🚀 快速开始

### 环境要求

- Python >= 3.9
- pip 包管理器
- 网络连接（访问天天基金 API）

### 安装步骤

```bash
# 1. 进入项目目录
cd fund_agent

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置 LLM API Key（推荐通义千问）
export LLM_API_KEY="sk-your-api-key"
export LLM_API_BASE="https://dashscope.aliyuncs.com/compatible-mode/v1"
export LLM_MODEL="qwen-plus"

# 4. 启动控制台对话（不需要配置文件）
python start.py
```

### 环境变量说明

| 环境变量 | 必填 | 说明 |
|----------|------|------|
| `LLM_API_KEY` | ✅ | LLM API 密钥 |
| `LLM_API_BASE` | ❌ | API 地址，默认通义千问 |
| `LLM_MODEL` | ❌ | 模型名，默认 qwen-plus |
| `LLM_TEMPERATURE` | ❌ | 温度参数，默认 0.5 |
| `WX_APP_ID` | 微信模式必填 | 微信测试号 appID |
| `WX_APP_SECRET` | 微信模式必填 | 微信测试号 appSecret |
| `WX_TOKEN` | 微信模式必填 | 微信测试号 Token |

#### 准备工作：部署到 Railway（获得公网 URL）

Railway 是一个云部署平台，免费额度足够运行本项目。

```bash
# 1. Fork 本项目到你的 GitHub
# 2. 注册 Railway 账号（railway.app），用 GitHub 登录
# 3. 在 Railway Dashboard 点击「New Project」→「Deploy from GitHub repo」
#    选择你 fork 的项目
# 4. 在 Railway 项目后台设置以下环境变量：
#    - WX_APP_ID       （微信测试号 appID）
#    - WX_APP_SECRET   （微信测试号 appSecret）
#    - WX_TOKEN        （自定义，如 fundagent123）
#    - LLM_API_KEY     （通义千问或其他 LLM 的 API Key）
#    - LLM_API_BASE    （可选，默认通义千问）
#    - LLM_MODEL       （可选，默认 qwen-plus）
# 5. 部署完成后，Railway 会给你一个公网域名，如：
#    https://fund-agent-production.up.railway.app
```

> 本项目自带 `railway_start.py`，Railway 会自动检测并运行，无需额外配置。

## 🗄️ 数据库表结构

项目使用 **TiDB Serverless**（兼容 MySQL 协议）作为数据持久化层。首次连接时自动创建以下 4 张表：

### 1️⃣ `user_portfolios` — 用户持仓表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `BIGINT` | **PK** AUTO_INCREMENT | 自增主键 |
| `user_id` | `VARCHAR(50)` | **UNIQUE**(user_id, fund_code) | 用户标识 |
| `fund_code` | `VARCHAR(10)` | | 基金代码 |
| `fund_name` | `VARCHAR(100)` | | 基金名称 |
| `channel` | `VARCHAR(50)` | | 存储渠道（支付宝/天天基金/银行等） |
| `cost_amount` | `DECIMAL(20,4)` | NOT NULL | **用户投入成本（元）** |
| `current_value` | `DECIMAL(20,4)` | DEFAULT 0 | **当前市值（元）** |
| `profit_rate` | `DECIMAL(10,4)` | DEFAULT 0 | **盈亏率（%）** |
| `shares` | `DECIMAL(20,4)` | NOT NULL | **自动计算的持有份额** |
| `updated_at` | `TIMESTAMP` | ON UPDATE NOW | 更新时间 |

### 2️⃣ `fund_basic_info` — 基金基本信息表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `fund_code` | `VARCHAR(10)` | **PK** | 基金代码 |
| `fund_name` | `VARCHAR(100)` | NOT NULL | 基金名称 |
| `fund_type` | `VARCHAR(50)` | | 基金类型 |
| `fund_company` | `VARCHAR(100)` | | 基金公司 |
| `manager_name` | `VARCHAR(50)` | | 基金经理 |
| `establish_date` | `DATE` | | 成立日期 |
| `total_size` | `DECIMAL(20,4)` | | 基金规模（亿元） |

### 3️⃣ `fund_nav_history` — 基金历史净值表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `BIGINT` | **PK** AUTO_INCREMENT | 自增主键 |
| `fund_code` | `VARCHAR(10)` | **INDEX** | 基金代码 |
| `nav_date` | `DATE` | **UNIQUE**(fund_code, nav_date) | 净值日期 |
| `unit_nav` | `DECIMAL(10,4)` | NOT NULL | 单位净值 |
| `acc_nav` | `DECIMAL(10,4)` | | 累计净值 |

### 4️⃣ `fund_returns` — 基金收益率表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `BIGINT` | **PK** AUTO_INCREMENT | 自增主键 |
| `fund_code` | `VARCHAR(10)` | **UNIQUE** | 基金代码 |
| `return_1m` | `DECIMAL(10,4)` | | 近1月收益率（%） |

### 持仓管理流程（无需配置文件）

```
用户通过微信/控制台说：
  "我在支付宝买了易方达优质精选，投入了2万块"
                  │
                  ▼
┌─────────────────────────────────────┐
│  1. 获取基金最新净值                  │
│  get_fund_realtime_estimate(code)   │
│  → 净值 = 2.5000                     │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  2. 自动计算持有份额                  │
│  shares = 20000 / 2.5000 = 8000份   │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  3. 存入数据库                        │
│  user_portfolios:                    │
│  cost_amount = 20000 (投入)          │
│  current_value = 20000 (初始市值)    │
│  profit_rate = 0.00 (初始盈亏率)     │
│  shares = 8000 (自动计算)            │
│  channel = "支付宝" (存储渠道)       │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  4. 查询时自动刷新市值和盈亏率         │
│  最新净值 × 份额 = 最新市值           │
│  盈亏率 = (市值 - 投入) / 投入 × 100% │
└─────────────────────────────────────┘
```
